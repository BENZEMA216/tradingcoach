## 2. P1 — 数据矛盾 / 功能失效

### 2.1 AI Coach 总盈亏数字矛盾（$1,035 vs $17,075）

- **症状**: AI Coach 页面同一屏显示两个不同的"总盈亏"
  - "AI 复盘总结": `盈利 $17,075`
  - "关键指标 → 盈亏": `$17,075.05`
  - "单笔风险过大": `占总盈利 $1,035 的 131%`
- **vs Dashboard/Statistics**: 都显示 `$1,035.42`
- **推测根因**: $17,075 是"盈利交易的总和"（只算正数），$1,035 是"净盈亏"（盈亏之和）。两者都叫"盈亏"，label 误导
- **建议**: AI Coach 的 "盈亏" 改成 "总盈利"，新增"净盈亏 $1,035.42" 字段；或直接统一显示 `net_pnl`

### 2.2 "策略分布 / 策略表现" 全部 Unclassified（100%）

- **Dashboard**: "策略分布" 饼图 100% Unclassified
- **Statistics**: "策略表现" 也只有"未分类 +$1,035.42"
- **根因**: 525 个仓位中 `strategy_type` 字段全部为 null（API 验证：`"strategy_type":null`）
- **影响**: 策略类 stats 全部失效
- **修复方向**: 后端策略自动分类逻辑未实现 / 未运行；或前端将 100% Unclassified 时降级为"暂未分类"空状态

### 2.3 "AI 服务不可用" 但仍显示 AI 总结

- **症状**: AI Coach 页顶部红字"AI 服务不可用"，但下方"AI 复盘总结" 显示了完整 472 笔/胜率/优势标的等内容
- **推测**: LLM 调用失败 → 回退到本地模板生成；但 UI 没区分"LLM 输出"vs"规则生成"
- **建议**: 模板生成的 insight 不该显示在 "AI 复盘总结" 区块下，或加 `[规则生成]` 标签

### 2.4 RelatedPositionsTab 内 PositionCard 货币硬编码 'USD'

- **症状**: `RelatedPosition` 类型缺 `currency` 字段，所以每张关联交易卡的金额永远以 USD 显示
- **影响**: 当关联交易包含港股（HKD）时，金额前缀会错（应是 `HK$` 而非 `$`）
- **修复方向**: 后端 `/related-positions` 返回包含 `currency`；前端类型加字段 + Card 内使用真实 currency

## 3. P2 — UI / 警告

### 3.1 Recharts 维度警告（仍存在）

- **Statistics 页**: console 警告 18 次 `width(-1) and height(-1) of chart should be greater than 0`
- **Dashboard 页**: 4 次同样警告
- **原因**: 即使加了 `minHeight={200}`，部分图表容器初始仍为 `-1×-1`（推测是 useElementSize 在首次 render 给的 -1）
- **修复方向**: ResponsiveContainer 包裹外层加 `width: 100%; min-height: 200px;`，或在容器 useEffect 完成前隐藏 chart

### 3.2 Events 页空状态体验差

- **症状**: market_data 为空时 Events 页显示"0 事件 / 暂无事件"
- **缺**: 没告诉用户"事件依赖市场数据，请先在 System 页导入"
- **修复**: 检测到 `market_data.count === 0` 时显示"请先导入市场数据"CTA

### 3.3 PositionDetail 价格走势 heading 重复

- **症状**: snapshot 显示两个 "价格走势" h3 紧挨在一起
- **可能原因**: 父子组件都渲染了标题

### 3.4 PositionDetail 总费用矛盾

- **交易摘要 → 总费用**: `$10.88`
- **执行 → 总费用**: `-$0.00`（Execution Tab 内计算）
- **每笔成交 → 费用**: `-`（占位符）

- **修复方向**: 后端 trade.total_fees 字段补齐，或 ExecutionTab fallback 用 position.total_fees

### 3.5 "待解决问题 17" 重复展示

- **Statistics 页**: AI 教练区块显示"17 待解决问题" 和 "17 问题"（同样数字两次）

### 3.6 持仓详情等级带"?"后缀

- **修复**: caption 改用绝对金额，或限制 max%

### 4.3 "最差时段 -511.79" 缺 "$" 前缀

- Statistics → 小时表现 → `最差: 22:00 (-511.79)`

### 4.4 做多盈亏 / 做空盈亏 缺统一正号

- 做多 `459 笔 -$1,855.13`（带负号 ✅）
- 做空 `13 笔 $2,890.55`（**缺正号**）
- **修复**: 都加 `+` / `-` 前缀，或用 `formatPnL`

## 6. Counterfactual 引擎信号（业务洞察）

测试中 Backtest 页揭露了重要信号，值得在产品定位中突出：

| 规则 | 跳过 | 多赚 | 多赚 % |
|------|------|------|-------|
| 严格 -10% 止损 | 107 笔 | **+$15,302** | **+1478%** |
| 规避长期亏损标的（FIG/NVDA spread/TQQQ/TSLA）| 68 笔 | +$6,005 | +580% |
| 反报复交易（亏后 2h 内不开新仓）| 334 笔 | +$2,246 | +217% |
| 仓位上限 3× 中位数 | 73 笔 | +$602 | +58% |
| 连亏熔断 | 157 笔 | +$299 | +29% |

**含义**: 用户实际盈亏 $1,035；如果当时严格执行 -10% 止损，盈亏将变成 $16,337。这是 `/复盘研究与Feature路线图.md` v2 中 G4 "错误价签" 的最佳锚定数据。
