## 二、问题清单

### P0 - 阻断级问题

#### P0-001: Sidebar 移动端适配缺失
**页面**: 全局布局
**文件**: `frontend/src/components/layout/Sidebar.tsx:27`
**描述**: Sidebar 固定 `w-64` 宽度，在移动端（<768px）会占据大部分屏幕，无汉堡菜单折叠功能。
**影响**: 移动端用户无法正常使用应用
**建议**:
- 添加响应式断点，移动端默认收起
- 添加汉堡菜单触发器
- 支持滑动手势展开/收起

---

### P1 - 体验级问题

#### P1-001: Landing 页面大量内容未翻译
**页面**: `/` (Landing)
**截图**: `01-landing-zh-i18n-issues.png`
**描述**: 切换到中文后，以下内容仍为英文：
- "PURE ALPHA." 主标语
- "Construct your edge with institutional-grade precision metrics..."
- "SYSTEM OPERATIONAL • V0.9.2"
- "UPLOAD PROTOCOL"
- "DRAG BINARY CSV HERE"
- "BROWSE DATA SOURCE"
- "EXECUTE ANALYSIS"
- 三个功能模块标题和描述

**建议**: 在 `frontend/src/i18n/locales/zh.ts` 中补充翻译

#### P1-002: 主题切换按钮文本未翻译
**页面**: 全局 Sidebar
**描述**: "Switch to Dark Mode" / "Switch to Light Mode" 在中文环境下未翻译
**建议**: 翻译为 "切换到深色模式" / "切换到浅色模式"

#### P1-003: 表格中 LONG/SHORT 未翻译
**页面**: `/positions` (持仓列表)
**描述**: 筛选器中正确显示"做多/做空"，但表格数据列仍显示 "LONG/SHORT"
**建议**: 统一翻译表格中的方向显示

#### P1-004: 持仓详情页 Back 按钮未翻译
**页面**: `/positions/:id`
**描述**: 返回按钮显示 "Back" 而非 "返回"
**建议**: 使用 i18n 翻译

#### P1-005: Info 按钮标签中英混用
**页面**: `/positions/:id`
**描述**: "Info about 持仓周期" 混用中英文
**建议**: 全部翻译为中文，如 "关于持仓周期的说明"

#### P1-006: Statistics 页面部分内容未翻译
**页面**: `/statistics`
**描述**: 以下内容未翻译：
- "Stock" (应为"股票")
- "Same Day", "1-3 Days", "4-7 Days" 等持仓周期标签
- "TOP 10 标的" 中的 "TOP 10"
- "[ HIDE ]", "[ SHOW ]" 按钮文本

**建议**: 补充 i18n 翻译

#### P1-007: 章节编号跳过 05
**页面**: `/statistics`
**描述**: 报告章节从 04/ PORTFOLIO 直接跳到 06/ DETAILED DATA，缺少 05/
**分析**: 05/ 是 EVENT ANALYSIS，可能因为没有事件数据而不显示
**建议**: 动态调整章节编号，或保留占位符

#### P1-008: 时间筛选器浅色模式对比度问题
**页面**: `/statistics`
**文件**: `frontend/src/pages/Statistics.tsx:365`
**描述**: 周期选择按钮使用 `bg-white/10`，在浅色模式下几乎不可见
**建议**: 为浅色模式添加适当的背景色，如 `bg-neutral-100`

#### P1-009: 文件选择后通知面板样式不协调
**页面**: `/` (Landing)
**文件**: `frontend/src/pages/LandingUpload.tsx:373-419`
**描述**: 选择文件后弹出的邮件/通知面板使用 `bg-white dark:bg-neutral-900`，与页面纯黑背景不协调
**建议**: 调整为与页面一致的工业风格设计

#### P1-010: Select 下拉箭头缺失
**页面**: `/positions`
**文件**: `frontend/src/pages/Positions.tsx:246`
**描述**: 使用 `appearance-none` 移除原生样式后，未添加自定义下拉箭头图标
**建议**: 添加自定义 chevron-down 图标

#### P1-011: 表单标签字号过小
**页面**: `/positions`
**描述**: 筛选面板标签使用 `text-[10px]`，在某些设备上可读性差
**建议**: 增大到 `text-xs` (12px)

#### P1-012: Loading 状态样式不统一
**页面**: 多处
**描述**: 各页面加载动画样式略有差异
**建议**: 创建统一的 Loading 组件

### P2 - 优化建议

#### P2-001: "NEURAL PYSCH" 拼写错误
**页面**: `/` (Landing)
**描述**: "PYSCH" 应为 "PSYCH" (Psychology 缩写)
**建议**: 修正拼写

#### P2-002: 注释风格副标题可能困惑用户
**页面**: `/positions`
**描述**: "// TRACKING_ID: 444 // 持仓记录" 编程注释风格对普通用户不友好
**建议**: 改为更友好的格式，如 "共 444 条记录"

#### P2-003: 颜色语义不一致
**页面**: `/positions`
**描述**: Long 使用蓝色，Short 使用紫色，可能与盈亏颜色（绿/红）产生认知冲突
**建议**: 考虑使用更中性的颜色区分方向

#### P2-004: 日期格式本地化
**页面**: 多处
**描述**: 使用 "2025/10/21" 格式，中文环境可考虑 "2025年10月21日"
**建议**: 根据语言环境动态格式化日期

#### P2-005: 主题切换按钮缺少 ARIA 标签
**页面**: `/` (Landing)
**描述**: uid=1_4 按钮没有可见文本标签，影响屏幕阅读器用户
**建议**: 添加 `aria-label`

#### P2-006: AI 教练数据不一致
**页面**: `/statistics`
**描述**: 01810 在 Top 10 表格显示 $7,282.31，在优势标的显示 $56,893
**建议**: 检查数据计算逻辑

#### P2-007: 缺少键盘导航支持
**页面**: 全局
**描述**: 部分交互元素无法通过 Tab 键访问
**建议**: 检查并修复 Tab 顺序和焦点样式

#### P2-008: 动画过渡缺失
**页面**: 多处
**描述**: 部分状态变化缺少过渡动画，体验生硬
**建议**: 添加适当的 transition 效果

## 五、总结

TradingCoach 整体设计质量较高，工业风格独特且专业。主要问题集中在**国际化翻译不完整**，这是最需要优先解决的问题。移动端适配是另一个重要问题，建议在下一版本中重点解决。

视觉设计和数据可视化方面表现优秀，AI 教练功能具有差异化价值。建议在修复翻译问题后，可以考虑将其作为产品亮点进行推广。
